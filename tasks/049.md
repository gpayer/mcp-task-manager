Prevent `tasks/.index.json` from being created when `mcp-task-manager` starts as an MCP server in a repository with no active tasks. The index file should only be written after the first task is created.

Implementation plan: `docs/plans/2026-03-26-lazy-index-creation.md`

Root cause summary:
- MCP startup always calls `Service.Initialize()`.
- `Initialize()` always calls `Index.Load()`.
- `Index.Load()` rebuilds when `.index.json` is missing.
- `Index.Rebuild()` currently saves an empty index even when there are no task markdown files.

Expected outcome:
- Starting the MCP server in an empty repo does not create `tasks/.index.json`.
- Creating the first task still writes both the task markdown file and the index file.
- Regression tests cover missing-index and empty-project startup behavior.
